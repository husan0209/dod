# docs/backup-restore.md

## Бэкапы и восстановление DOD Platform

### 1. Расписание бэкапов
- Каждые 6 часов автоматический бэкап
- Ручной бэкап перед деплоем

### 2. Хранение
- Локально: /opt/dod/deploy/backups/data
- S3: для долговременного хранения

### 3. Retention
- 30 дней хранения бэкапов
- Автоматическое удаление старых

### 4. Процедура восстановления
- Остановить приложение
- Восстановить базу из dump
- Запустить приложение
- Проверить данные

### 5. Тестирование бэкапов
- Ежемесячно тестировать восстановление
- Проверять целостность данных

### 6. Disaster Recovery план
- RTO: 30 минут
- RPO: 6 часов
- Процедура: новый сервер, восстановить бэкап, DNS update
